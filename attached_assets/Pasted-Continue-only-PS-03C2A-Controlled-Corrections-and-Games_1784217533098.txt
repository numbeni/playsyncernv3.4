# 1. Fix the Game creation regression

Observed behavior:

In the “Add New Game” modal, a valid title such as `gta` is entered, a valid platform and status are selected, but after clicking “Add Game” the UI displays:

بازی مورد نظر یافت نشد

This message is incorrect for a Game creation request.

Inspect the complete flow:

- frontend form submit handler
- mutation request URL and HTTP method
- API base path
- POST `/games` route
- response status and response body
- returned Game ID
- cache invalidation/refetch logic
- any automatic navigation or follow-up GET request
- frontend API-error mapping

Determine whether the 404 is returned by:

- the POST request itself
- an incorrect route
- a follow-up request using a missing or incorrect Game ID
- stale cache/refetch behavior
- incorrect interpretation of another API error

Required behavior:

- submitting valid Game data creates exactly one Game
- the modal closes only after successful creation
- the Games list refreshes and displays the created Game
- no incorrect “Game not found” message is shown
- 404 is used only when an existing Game ID genuinely does not exist
- duplicate title returns the correct duplicate/conflict message
- validation errors display the correct validation message
- the submit button is disabled while the request is pending
- repeated clicks must not create duplicate requests
- on failure, entered form values remain available for correction

Do not solve the issue by hiding the error message or hard-coding a successful response.

Add or update tests covering:

- successful Game creation
- created Game appears in the list
- duplicate title handling
- failed creation does not close the modal
- no duplicate request during pending submission
- create flow never maps a normal POST failure to “Game not found” incorrectly

Preserve all existing Games behavior.
